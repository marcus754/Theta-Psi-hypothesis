# AGENTS.md

## Цель
Поддерживать гипотезу Θ–Ψ как исследовательский проект: сначала смысл и формулы в документах, затем проверка скриптами.

## Правила работы
1. `docs/notes/смысл.md` не изменять и не удалять без явной команды пользователя.
2. Скрипты и код — инструмент проверки; первичный артефакт для людей — документы в `docs/`.
3. Не раздувать структуру: избегать дублей и лишних `README`; объединять материалы по смыслу.
4. Любое изменение, влияющее на смысл формул, синхронизировать между `docs/` и `background/`.
5. Перед удалением файлов проверять ссылки по репозиторию.

## Минимальная структура notes
- `docs/notes/смысл.md` — авторские формулировки.
- `docs/notes/core.md` — единый рабочий конспект (ядро, план, ориентиры).
- `docs/notes/frw_reduced_eom.md` — приведенные фоновые уравнения.
- `docs/notes/STATUS.md` — статусы и каноничность артефактов.

## Формат коммуникации
- Писать по-русски.
- По делу, без канцелярита. 
- Не ужимать смысл. Читатель должен легко понимать написанное без домысливания.
- После правок сообщать: что изменено, где, и зачем.
