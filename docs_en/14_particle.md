14. PARTICLE AS A DERIVATIVE OBJECT

DEFINITION

In the current Theta–Psi framework, a particle is not introduced as a third primary entity.
Canonical formulation:
particle = stable localized Theta-projection of a Psi-sector.

That is, a particle is a stable way in which a Psi-sector manifests via Theta as a localizable event with repeatable quantum labels.
Only the following remain primary:
Theta_munu, Psi.

ACCEPTANCE CRITERIA

A derived particle sector passes the minimal check only if simultaneously:
1. there is a localized finite footprint:
finite norm,
finite width,
finite energy.
2. labels identifying the repeatable sector are preserved:
mass, charge, spin.
3. there is a detector event map:
local Psi-density + detector coupling -> Theta-local detector event.
4. product and entangled multiparticle sectors are distinguished:
Psi_AB = Psi_A ⊗ Psi_B -> independent complete states,
Psi_AB != Psi_A ⊗ Psi_B -> one nonseparable Psi-sector with multiple Theta-events.
5. a classical trajectory limit exists:
S_action / hbar >> 1,
decoherence_strength > 0.

MEANING

A particle does not have to be a small object that always has all properties in space. In Theta–Psi, it is a stable class of manifestations:
Psi-sector
  -> Theta-localization anchor
  -> detector event
  -> conserved labels
  -> repeatable particle identity.

For an entangled pair:
Psi_AB != Psi_A ⊗ Psi_B,
local Theta-events = 2,
independent complete states = 0.

This is consistent with the EPR layer: two registrations can be local events but do not have to be two complete independent states.
