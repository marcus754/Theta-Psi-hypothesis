# 03. Скалярные возмущения (ADM)

## Канон времени

- Фундаментальный объект времени в гипотезе: тензор Θ_{μν} (см. docs/11_derivation_path.md).
- В формулах этого раздела используется редуцированная FRW-переменная θ ≡ θ_eff, где θ_eff^2 = Θ_{μν} u^μ u^ν, u^μ u_μ = -1.
- Все записи вида n(θ) в этом файле следует читать как n(θ_eff, Θ, Ψ).
- Этот раздел описывает вычислительный редуцированный контур, а не полную тензорную динамику Θ_{μν}.

## Каноническая постановка (обязательно для всех разделов)

В проекте базой принимаются Θ_{μν} и Ψ; редуцированный контур не требует внешнего порога и не использует ручное отсечение веток.
S = ∫ d^4x √(-g) [ (M_pl^2/2)R + L_m + L_{ΘΨ}
                   + M_*^4 F(J_refr) ],
J_refr(Θ,Ψ)=Φ_eff(θ_eff)·I_grad(Θ,Ψ).

Источник режима (без Weyl-порога в каноне):
I_grad(Θ,Ψ) = l_θ^2 · |∇_⊥θ|^2 + l_Ψ^2 · |∇_⊥Ψ|^2, ∇_⊥ = пространственная часть градиента (ортогонально локальному u^μ).

Смысл:
- В FRW (однородный фон): I_grad=0, поэтому J_refr=0.
- В strong-field (большие пространственные градиенты): I_grad>0, поэтому преломляющий сектор включается напрямую через F(J_refr).
- Отдельного режимного поля нет; канонический scalar/ADM-контур содержит только возмущения редуцированных полей δθ, δR, δφ.

Индекс вакуума задаётся конечной рабочей формой:
n(θ_eff,Θ,Ψ)=1+𝒩(J_refr),
𝒩(0)=0,\quad 𝒩(J)\ge 0,\quad 𝒩(J)\ \text{монотонна и конечна}.

Невалидная ветка фона исключена параметризацией (а не фильтром):
A = 3 tanh^2( sqrt(A_raw/3) ) in [0,3),
H^2 = (ρ_bg + V/2 + ρ_phase)/(3-A).

В сравнении с GR проверяется не «ручное включение», а прямой ответ на пространственные неоднородности Θ_{μν}, Ψ.

### Синапсис
- Пертурбации q=(δθ, δR, δφ). Квадратичное действие: L^(2)=½ δq̇ᵀKδq̇ − ½ δqᵀ[ G (k²/a²) + M ]δq.
- Минимальная high‑k модель: K=G=diag(γ²+3/θ², 1, R²) ⇒ скалярные моды распространяются со световой скоростью (c_s=1).
- Массы: M_RR = V0 + 3 Q²/(a⁶ R⁴); M_θθ = M_φφ = 0; k⁰‑поправки ΔM_ab(H, Ḣ, θ̇, Ṙ, Q).
- Исключение δN,B через Шур: A_eff = A − B C⁻¹ Bᵀ; восстанавливаем G_eff, M_eff.
- Условия здоровья: K≻0; eig(K⁻¹G_eff)>0 (high‑k); ω²(k)>0 (finite‑k); запрет сверхсвета.


Цель: матрицы K(a,k), G(a,k), M(a,k) для (δθ, δR, δφ) после исключения алгебраических ADM‑переменных (lapse/shift), с возможностью проверять призраки/градиенты и конечнок‑волновую дисперсию ω²(k).

Реализация
- Эффективное действие без ограничений:
  L^(2) = 1/2 · δq̇ᵀ K δq̇ − 1/2 · δqᵀ [ G (k²/a²) + M ] δq,   δq = (δθ, δR, δφ).

- Исключение lapse/shift (алгебраические переменные c = (δN, B)) вводим в виде
  L^(2) ⊃ − δqᵀ B c − 1/2 · cᵀ C c.

  Стационарность по c даёт c* = − C⁻¹ Bᵀ δq и эффективный блок
  A_eff = A − B C⁻¹ Bᵀ,  где A = G (k²/a²) + M.

  Затем восстанавливаем G_eff и M_eff как части A_eff, пропорциональные (k²/a²) и константной, соответственно.
  В минимальном строгом режиме используем эквивалентную форму без явных смешений: K=diag(γ²+3/θ²,1,R²), G=K (люминально),
  M = M_pot + ΔM_grav(k⁰). ΔМ реализуется по умолчанию через символический Schur: M_eff = M_pot − B C⁻¹ Bᵀ, где
  B=[G·φ̇, D_t(G·φ̇)], а C^{-1}=[[3H−Ḣ/H, 1/H],[1/H, 0]]. Старые API с именем sigma_strict считаются legacy-именами этого блока.

- Диагональный блок теперь рассматривается как минимальная согласованная реализация текущего действия, а не аппроксимацией для подгонки:
  K=G=diag(γ²+3/θ²,1,R²),
  M=diag(0,V0+3Q²/(a⁶R⁴),0).
  Поэтому в каноне ε_K=ε_G=α_k²=0. Ненулевые смешения допустимы только как диагностический вспомогательный режим или после добавления явных операторов в действие.
- Численно generalized eigenvalues считаются через холецки‑редукцию (симметризованный K⁻¹G), см. src/linear_modes_adm.py.

Рабочие выражения (диагональная аппроксимация, high‑k):

- Кинетика K
  - Kθθ = 3/θ² + γ²
  - KRR = 1
  - Kφφ = R²

- Градиенты G (коэффициенты при k²/a²)
  - Gθθ = Kθθ (минимальная люминальная модель, c_s=1)
  - GRR = 1
  - Gφφ = R²

- Массы M
  - Mθθ = 0 (по умолчанию)
  - MRR = V0 + 3 Q²/(a⁶ R⁴)
  - Mφφ = 0

Интерфейс
- Символьные/аналитические выражения K,G,M заданы в theory/adm_symbolic.py. Старое имя elim='sigma_strict' является legacy-режимом для минимального Schur-блока.
- Быстрые проверки и собственные значения реализованы в src/linear_modes_adm.py:
  - cs2_eigs_adm(...) — c_s² как собственные значения обобщённой задачи G v = λ K v.
  - dispersion_eigs_adm(...) — ω²(k) из K⁻¹ [ G (k²/a²) + M ].
  - stability_adm(...), stability_finite_k_adm(...) — флаги K≻0, c_s²>0, ω²>0.
  - Везде доступен параметр elim={...} для включения исключения lapse/shift.

Дальнейшая работа
- Заменить диагональную аппроксимацию на полный вывод с метрикой (появятся контролируемые смешения K_ij, G_ij, M_ij).

### Практические замечания (строгий профиль)

- В проверках здоровья при use_adm=True по умолчанию включается при конечных k (ω²>0) вдоль траектории.
- Строгий Schur-режим (elim='sigma_strict', legacy name) не использует эвристические «дефолтные» исключения; только K=G и k⁰‑поправки к массам.

## Аналитические условия (аппроксимация)

- Призраки (K≻0): без смешений достаточно Kθθ>0, KRR=1, Kφφ=R²>0. Со смешениями ε_K — выполняются условия Сильвестра для 2×2 и 3×3 миноров.
- Градиенты: собственные значения K⁻¹G положительны; без смешений это требования c_θ²,c_R²,c_φ²>0. Субсветовость реализуется как c_i²≤1.
- Конечные k: ω² — собственные K⁻¹ [ G (k²/a²) + M ]. При V0≥0 и положительных c_i² получаем ω²(k)>0 при k>0.

Практическая область (демо): c_i²∈(0,1], |ε_K|,|ε_G|≪1, V0∈[1e−3,1]. См. также 05_consistency_checks.md.

Замечание о гиперболичности и субсветовости
- Гиперболичность обеспечивается положительной определённостью K и положительностью собственных значений K⁻¹G (high‑k) и K⁻¹[ G(k²/a²)+M ] (finite‑k).
- Субсветовость задаётся требованием max eig(K⁻¹G) ≤ 1 (с малым tol). Эти условия соответствуют реализованным в коде проверкам.

### Гравитационные k^0‑поправки

Для минимальной модели (G_ab=diag(γ²+3/θ²,1,R²)) после исключения δN,B появляется k^0‑вклад в масс‑блок:
ΔM_ab = − (3H − Ḣ/H) T_ab − (1/H) D_t T_ab,
T_ab = φ̇_a φ̇_b,   φ̇_a = G_ab φ̇^b,
φ̇^θ = θ̇,
φ̇^R = Ṙ,
φ̇^φ = Q/(a³ R²).

Итоговый M = M_pот + ΔM с M_RR^pot = V0 + 3Q²/(a⁶R⁴). В коде: theory/adm_symbolic.mass_matrix_sigma_total.
