# 10. Фальсифицируемые критерии


## Канон времени

- Фундаментальный объект времени в гипотезе: тензор Theta_munu (см. docs/11_derivation_path.md).
- В формулах этого раздела используется редуцированная FRW-переменная theta ≡ theta_eff, где theta_eff^2 = Theta_munu u^mu u^nu, u^mu u_mu = -1.
- Все записи вида n(theta) в этом файле следует читать как n(theta_eff, Theta, Psi).
- Этот раздел описывает вычислительный редуцированный контур, а не полную тензорную динамику Theta_munu.

## Каноническая постановка (обязательно для всех разделов)

В текущем каноне первичны только две сущности: Theta_munu и Psi. Сильное поле отклик задаётся напрямую составным инвариантом режима

J_refr(Theta,Psi) = Phi_eff(theta_eff) · I_grad(Theta,Psi),
I_grad(Theta,Psi) = l_theta^2 · |∇_⊥theta|^2 + l_Psi^2 · |∇_⊥Psi|^2.

Рефрактивный сектор задаётся через

L_refr = M_*^4 F(J_refr),
n = n(theta_eff, Theta, Psi),
g_hat_munu = g_munu + (n^2 - 1) u_mu u_nu.

Ключевое разделение для этого файла:
- empirical проверки опираются только на прямые data-facing наблюдаемые и используются в scripts/run_falsifiability_check.py;
- diagnostic проверки могут использовать внутренние диагностики, сканs и калибровочные артефакты, но не считаются жёстким критерием фальсификации.

## 0) Порядок работы

В текущей версии уже зафиксированы:
- единый редуцированный канон;
- ковариантная форма действия;
- связь фонового и сильнопольного режимов;
- скалярный сектор возмущений;
- базовый набор проверок устойчивости.

Незавершёнными остаются:
- полная динамика тензорного временного поля;
- общий нестационарный сильнопольный режим;
- строгий вывод всех квантовых эффектов из базового действия.

## 1) Пакет слабопольных тестов

Критерии проверяются через scripts/run_prediction_suite.py и scripts/run_falsifiability_check.py.

- Меркурий (аномальная прецессия):
  - целевое значение 42.98"/век,
  - допустимое отклонение: |pull| <= 3σ.
- Красное смещение на поверхности Солнца:
  - диапазон 1.8e-6 .. 2.4e-6.
- Шапиро (roundtrip, near conjunction):
  - диапазон 1.0e-4 .. 4.0e-4 секунд.
- Отклонение света у лимба Солнца:
  - диапазон 1.70 .. 1.80 угл. сек.

## 2) Критерий по совместной области

По scripts/run_joint_region_скан.py:

- должна существовать ненулевая 3σ-область (N_3sigma >= 1),
- при этом 3σ-область не должна занимать почти всю сетку (f_3sigma <= 0.20) — иначе модель нефальсифицируема на данном наборе тестов.

## 3) Diagnostic сильное поле сканs

Сканы вроде scripts/run_compact_constraint_скан.py и inferred-переходы из strong_field_bounds_registry.json остаются в репозитории только как архивная диагностика. Они могут использовать inferred-переходы вида наблюдение -> inferred индикатор, поэтому не входят в PASS/FAIL по falsifiability.

## 4) Машиночитаемая спецификация

Пороговые значения для empirical falsifiability вынесены в:
- fitting/data/prediction_цельs.json.

Автоматический чек:
- python scripts/run_falsifiability_check.py.

Для текущей версии канона приоритетны два класса прямых отличающих тестов:
1. сильное поле saturation tests на первичном compact-канале (z_surface, delay, ok_no_horizon), а ring_uas и
   echo_delay_ms использовать как downstream прямой checks;
2. background compatibility tests на CC+SN+BAO, но только как второй слой после закрытия пунктов 1-3 из раздела выше.

Если итог FAIL, версия считается незавершенной и требует доработки по empirical-контуру. Diagnostic-отчёты могут падать или расходиться без автоматического статуса фальсификации.
