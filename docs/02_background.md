# 02. Фон: единицы, лагранжиан, уравнения


## Канон времени

- Фундаментальный объект времени в гипотезе: тензор `Θ_{μν}` (см. `docs/11_derivation_path.md`).
- В формулах этого раздела используется редуцированная FRW-переменная
  `θ ≡ θ_eff`, где `θ_eff^2 = Θ_{μν} u^μ u^ν`, `u^μ u_μ = -1`.
- Все записи вида `n(θ)` в этом файле следует читать как `n(θ_eff)`.
- Этот раздел описывает вычислительный редуцированный контур, а не полную
  тензорную динамику `Θ_{μν}`.

## Каноническая постановка (обязательно для всех разделов)

Этот файл относится только к FRW-модулю. Он использует физический базис
гипотезы, но не заменяет собой ни онтологию времени, ни strong-field слой.

```
S = ∫ d^4x √(-g) [ (M_pl^2/2)R + L_m + L_{ΘΨ} + L_refr(Θ,Ψ) ].
```

В текущем каноне первичны только `Θ_{μν}` и `Ψ`. В полной записи `Ψ`
рассматривается как комплексный order parameter `Ψ = R e^{iφ}`; в FRW-редукции
используется амплитуда `R` и фазовый заряд `Q = a^3 R^2 \dot φ`, а `|Ψ|^2`
естественно читается как плотность амплитуды. Strong-field отклик
параметризуется напрямую составным инвариантом

```
J_refr(Θ,Ψ)=Φ_eff(θ_eff)·I_grad(Θ,Ψ),
I_grad(Θ,Ψ)=l_θ^2 · |∇_⊥θ|^2 + l_Ψ^2 · |∇_⊥Ψ|^2.
```

На однородном FRW-фоне:

```
I_grad = 0,   J_refr = 0.
```

Поэтому фоновая космология сама по себе не должна включать strong-field
рефрактивную ветвь. Рабочая форма индекса остаётся

```
n(θ_eff,Θ,Ψ)=1+𝒩(J_refr).
```

Невалидная ветка фона исключена параметризацией:

```
A = 3 tanh^2( sqrt(A_raw/3) ) in [0,3),
H^2 = (ρ_bg + V/2 + ρ_phase)/(3-A).
```

Раздел задает только базовые единицы, однородный лагранжиан и FRW-уравнения,
которые используются во всех численных проверках фонового модуля.

# 02. Единицы и масштабы

Базовая система: 8πG = 1, c = 1 (редуцированные планковы единицы). Уравнение Фридмана: H² = ρ_tot / 3.

H0 (70 км/с/Мпк)
- Перевод в безразмерные коды: используем `background.components.H0_from_km_s_Mpc`.
- Формула: `H0_dimless = (H0_km_s_Mpc · 1000 / Mpc_in_m) · t̄_P`, где
  - `Mpc_in_m = 3.085677581491367×10²² м`,
  - `t̄_P ≡ √(8π) t_P ≈ 2.70117733×10⁻⁴³ с` — редуцированное планково время.
- Пример: 70 км/с/Мпк → ≈ 6.13×10⁻⁶¹ (безразмерно в наших единицах).

Гамма γ
- В выбранных единицах γ — безразмерна. Физически допустимая область: γ ≲ 10⁻¹⁸.
- Демонстрации с γ ~ O(1) служат для проверки численной схемы и “прочистки труб”; в реалистичных прогонах используйте γ ≈ 10⁻¹⁸.
- Тест `tests/test_gamma_limit.py` удостоверяет, что при фиксированных физических нач. условиях фона H(a) инвариантен к выбору γ (Ψ=0, θ̇=0), что исключает “скрытое раздувание” эффектов выбором единиц.

Размерности (в выбранных единицах)
- θ — безразмерная; R (амплитуда Ψ) — безразмерный масштаб поля в коде; V0 — эффективная “жёсткость” (время⁻² в физ. единицах); Q — безразмерный интеграл движения.

Практика
- Для сопоставления с данными сначала приводите H0 к безразмерным единицам (`omegas_from_km_s_Mpc`), затем выбирайте γ ≈ 10⁻¹⁸ и сканируйте V0, Q под проверками здоровья.

## Реалистичный прогон (γ ≈ 10⁻¹⁸)

Шаги:
- Постройте фоновые параметры: `om = omegas_from_km_s_Mpc(70.0)`.
- Возьмите `γ = 10^{-18}`, `Q = 0` (для начала), выберите разумные начальные условия в ранней радиационной эпохе (например, `a0=1e-3`, `θ̇≈0`, `R≈0`).
- Интегрируйте фон по `N = ln a`: `integrate_background_lnN(a0, x0, xdot0, R0, Rdot0, N1, p)`.

Фоновая диагностическая величина:

```
E(a) = H(a)/H0.
```

Сопоставление этой величины с данными относится к observational-слою. Здесь
важно лишь, что FRW-модуль производит самосогласованный `H(a)` из редуцированной
динамики `Θ–Ψ`.

# 03. Однородный лагранжиан

Базовый однородный лагранжиан (minisuperspace) с полем времени θ(t) и комплексным субстратом Ψ(t) = R(t) e^{iφ(t)}:

```
L_eff = 1.5 (θ̇/θ)² + 0.5 Ṙ² + 0.5 R² φ̇² - 0.5 V0 R² + 0.5 γ² θ̇².
```

Комментарии
- Канонические знаки для R и φ.
- Удобно работать с интегралом движения Q = a³ R² φ̇ ⇒ φ̇ = Q/(a³ R²). Тогда фазовая часть в ρ,p: 0.5 Q²/(a⁶ R²).
- В коде (см. background/frw_symbolic.py) реализована часть без R²φ̇² (Q=0). Для Q≠0 фазовый вклад добавляется на уровне фона (см. раздел «04. Фоновые уравнения» ниже).

Энергия и давление (без общего множителя a³)
```
ρ = Σ_i (∂L/∂q̇_i) q̇_i - L,   p = L.
```

# 04. Фоновые уравнения

Переменные
- Масштабный фактор a(t), H = ȧ/a.
- Регуляризованная переменная x: θ = (√3/γ) tanh x, θ̇ = (√3/γ) sech² x · ẋ.
- Амплитуда R, фаза φ, заряд Q = a³ R² φ̇.

Оптическая переменная времени:

```
dτ = dt / n(θ_eff),  1 ≤ n(θ_eff) < ∞.
```

Это условие встроено в физическую постановку и исключает истинный горизонт событий:
для любых допустимых профилей θ конечны и `1+z`, и `∫ n dl`.

Уравнения движения
```
ẍ + 3H ẋ = ẋ² (2 tanh x + coth x)
R̈ + 3H Ṙ − Q²/(a⁶ R³) + V0 R = 0
φ̇ = Q / (a³ R²)
```

Энергия
```
ρ_fields = ρ(θ,θ̇,R,Ṙ; γ,V0) + 0.5 Q²/(a⁶ R²)   (фазовая добавка при Q≠0)
H² = (ρ_fields + ρ_m + ρ_r + ρ_Λ) / 3
```

Ограничения и безопасность
- Замена θ=(√3/γ) tanh x — численная регуляризация (без физического ограничения);
  прямой интегратор по θ доступен и даёт ту же физику, а возможные дивергенции при 1−γ²θ²/3→0
  являются следствием самих уравнений движения.
- При Q≠0 амплитуда R не может проходить через 0 (центробежный барьер).
- Ограничение на наблюдаемость компактных объектов задаётся конечностью
  redshift на реализованной strong-field ветке:
  `1+z = n_emit/n_obs < ∞`.

## Переменная N = ln a (для численной устойчивости)

Удобно интегрировать фон по e‑folds N = ln a. Обозначив y = dx/dN, u = dR/dN, получаем
алгебраическое выражение для H² без итераций:

```
A(x,y,u) = 3/2 · (sech⁴x / tanh²x) · y² + 1/2 · u² + 3/2 · sech⁴x · y²
H² = [ρ_m(a) + ρ_r(a) + ρ_Λ(a) + 1/2 V0 R² + 1/2 Q²/(a⁶ R²)] / (3 − A).
```

А затем

```
Ḣ = −1/2 · [ γ² θ̇² + Ṙ² + 3 (θ̇/θ)² + Q²/(a⁶ R²) + ρ_m + 4/3 ρ_r ]
```

что позволяет записать уравнения для y′(N), u′(N) и t′(N)=1/H. См. реализацию `integrate_background_lnN` в `background/frw_background.py`.

## Примечание о переменных

- x-переменная устраняет полюс в правой части уравнения для θ и повышает численную устойчивость.
- Для демонстрации отсутствия искусственных ограничений реализован прямой интегратор по θ:
  см. функции `rhs_theta` и `integrate_theta` в `background/frw_integrator.py`.
